Atividade Prática 4.1

Implemente, em R, um experimento computacional para um problema de classificação binária utilizando a base SAheart da biblioteca ElemStatLearn, a qual reúne informações clínicas e comportamentais associadas à ocorrência de doença coronariana. Nesta atividade, o objetivo em prever a presença de doença coronariana, representada pela variável resposta chd, em que \(y=1\) indica presença da doença (classe positiva) e \(y=0\) indica ausência (classe negativa). As variáveis explicativas disponíveis incluem: pressão arterial sistólica (sbp), consumo de tabaco (tobacco), colesterol LDL (ldl), adiposidade (adiposity), histórico familiar de doença coronariana (famhist), escore de personalidade tipo A (typea), obesidade (obesity), consumo de álcool (alcohol) e idade (age). Para isto, dado que a variável chd apresenta desbalanceamento moderado, com cerca de \(65\%\) de indivíduos sem doença e \(35\%\) com doença, será considerado o ajuste de um classificador baseado em regressão logística sob dois cenários: sem reamostragem (mantendo-se o desbalanceamento original) e com reamostragem via random undersampling (balanceamento no conjunto de treinamento). O procedimento experimental deve ser conduzido conforme as etapas a seguir:

  • Fixar uma semente aleatória para reprodutibilidade (por exemplo, set.seed(1212));

  • Carregar a base SAheart, definir a variável resposta chd como fator com níveis \(\{ 0,1\}\) e tratar famhist como variável categórica, mantendo as demais como quantitativas;

  • Particionar os dados em conjuntos de treinamento e teste utilizando o método hold-out, com \(70\%\) das observações destinadas ao treino e \(30\%\) ao teste, reportando a distribuição de classes em cada subconjunto;

  • Ajustar um modelo de regressão logística, via glm(), no conjunto de treinamento sem reamostragem, obter probabilidades previstas no conjunto de teste, convertê-las em classes preditas por meio de um ponto de corte fixo (por exemplo, \(0,5\)), e calcular a matriz de confusão e métricas de desempenho;

  • Aplicar random undersampling exclusivamente no conjunto de treinamento, reduzindo aleatoriamente a classe majoritária (\(y=0\)) até obter um conjunto aproximadamente balanceado (razão 1:1), e reportar a distribuição de classes após a reamostragem;

  • Ajustar novamente o modelo de regressão logística, via glm(), no conjunto de treinamento reamostrado e avaliá-lo no mesmo conjunto de teste, registrando matriz de confusão e métricas diretamente comparáveis às obtidas no cenário sem reamostragem.

Ao final, organize os resultados em um quadro comparativo e elabore uma síntese interpretativa que discuta (i) por que a acurácia pode se mostrar inflada em situações de desbalanceamento moderado, (ii) como o random undersampling reorienta o processo de ajuste do modelo em direção à classe positiva e (iii) quais métricas devem ser priorizadas quando o objetivo é a identificação de indivíduos com doença coronariana.

conf_mat        <- function(y_true, y_pred) 
{
  tab           <- table(Real = y_true, Pred = y_pred)
  all_levels    <- c("0", "1")
  tab2          <- matrix(0, nrow = 2, ncol = 2,
                          dimnames = list(Real = all_levels, Pred = all_levels))
  tab2[rownames(tab), colnames(tab)] <- tab
  tab2
}

metrics_cm      <- function(cm) 
{
  TN            <- cm["0", "0"]
  FP            <- cm["0", "1"]
  FN            <- cm["1", "0"]
  TP            <- cm["1", "1"]

  acc           <- (TP + TN) / sum(cm)
  sens          <- if ((TP + FN) > 0) TP / (TP + FN) else NA
  spec          <- if ((TN + FP) > 0) TN / (TN + FP) else NA
  prec          <- if ((TP + FP) > 0) TP / (TP + FP) else NA
  f1            <- if (!is.na(prec) && !is.na(sens) && (prec + sens) > 0) {
                      2 * prec * sens / (prec + sens)
                   } else NA

  c(Acuracia = acc, 
    Sensibilidade = sens, 
    Especificidade = spec,
    Precisao = prec, 
    F1 = f1)
}

  1. Acurácia inflada: Em cenários de desbalanceamento (como o de chd, onde a maioria não tem a doença), um modelo que simplesmente preveja sempre a classe majoritária (0) acertará na maior parte das vezes. Isso gera uma acurácia global alta, dando a falsa impressão de que o modelo é excelente, enquanto ele é péssimo em identificar a classe de interesse (os doentes).

  2. O papel do Random Undersampling: Ao remover aleatoriamente exemplos da classe majoritária até igualar à minoritária (1:1), nós forçamos o algoritmo a dar a mesma importância para ambas as classes. A fronteira de decisão da regressão logística é reorientada para não mais “favorecer” a classe majoritária. O modelo perde informações do conjunto geral, mas aprende melhor os padrões que definem a classe positiva.

  3. Métricas prioritárias: Para o diagnóstico de doença coronariana, o custo de um falso negativo (dizer que um doente está saudável) é muito mais grave do que um falso positivo (pedir exames extras para um paciente saudável). Logo, a Sensibilidade (Recall) deve ser a métrica priorizada, acompanhada do F1-Score, que nos ajuda a garantir que não estamos apenas classificando todo mundo como doente (equilíbrio com a Precisão).
